# EARS 需求工程标准化分享文档包

这套文档用于分享和实战落地 EARS（Easy Approach to Requirements Syntax）需求写法。

## 文件说明

- `ears-requirements-standard-wechat.md`  
  主文档：讲清楚 EARS 的定义、含义、句式、流程、技巧、质量门和落地建议。

- `ears-examples-library.md`  
  示例库：覆盖简单需求、复杂需求、权限、异常、状态机、非功能需求等场景。

- `ears-llm-conversion-playbook.md`  
  大模型转化手册：提供从会议纪要/PRD/工单转成 EARS 的流程、提示词和结构化输出方案。

- `assets/`  
  配图资源，包括 EARS 总览、句式模板、落地流程、大模型转换流水线、质量门。

## 建议阅读顺序

1. 先读主文档，建立 EARS 方法框架。
2. 再看示例库，学习不同场景怎么拆需求。
3. 最后看大模型转化手册，把方法放进日常需求整理流程。

## 适用对象

- 产品经理
- 需求分析师
- 研发负责人
- 测试工程师
- 项目经理
- 研发效能团队
- 希望用 AI 辅助需求工程的团队
